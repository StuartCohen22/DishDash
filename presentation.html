What is DishDash?

🥗

Recipe Management

Create and organize recipes with categorized ingredients, prep/cook times, and instructions.

📅

Weekly Meal Planning

Drag recipes onto a 7-day calendar grid, assigning breakfast, lunch, dinner, or snack slots.

🛒

Auto Shopping Lists

One click aggregates all ingredients from a week's meals, deduplicating and summing quantities.

🔐

Secure Auth

JWT-based authentication with bcrypt password hashing — all data scoped per user.

Database Schema

users

  • id (UUID PK)
  • email (unique)
  • password_hash
  • name
  • created_at

recipes

  • id (UUID PK)
  • user_id (FK →users)
  • name, description
  • instructions
  • prep/cook_time_minutes
  • servings, category

→ ingredients (cascade delete)

meal_plans

  • id (UUID PK)
  • user_id (FK →users)
  • name
  • week_start_date

→ planned_meals (day_of_week + meal_type)

→ shopping_lists (1:1)

shopping_items

  • ingredient_name
  • quantity (Numeric)
  • unit (enum)
  • is_purchased

Aggregated from all recipe ingredients in a plan

Enums:  RecipeCategory (breakfast · lunch · dinner · snack · dessert · beverage)  |  IngredientUnit (cup · gram · ounce · teaspoon · …12 values)  |  MealType (breakfast · lunch · dinner · snack)

Dependency Injection

CurrentUser is a FastAPI Depends that decodes the JWT, queries the user from DB, and injects it. Zero boilerplate per route.

Atomic Transaction

Recipe + all ingredients are written in the same SQLAlchemy AsyncSession. A rollback on any error leaves the DB clean.

Pydantic v2 Validation

FastAPI auto-returns 422 Unprocessable Entity with field-level errors if the request body fails schema validation.

Ownership Scoping

Every query filters by user_id — users can only read/modify their own data.

Feature Deep-Dive: Shopping List Generation

POST /api/v1/meal-plans/{id}/shopping-list

① Load Meal Plan

Eager-load: meal_plan → planned_meals → recipe → ingredients

② Aggregate Ingredients

Group by (name_lower, unit) — sum quantities using Decimal arithmetic

③ Pantry Detection

Flags common spices/small-quantity items (salt, pepper, vanilla…) as is_pantry_item

④ Persist & Return

Creates ShoppingList + ShoppingItem rows; frontend renders checkable list

📄 services/shopping.py — aggregation core

# Aggregate across all meals aggregated = defaultdict(list) for planned_meal in meal_plan.planned_meals: for ing in planned_meal.recipe.ingredients: key = (ing.name.lower(), ing.unit.value) aggregated[key].append(ing.quantity) # Create shopping items for (name, unit), qtys in aggregated.items(): total = sum(qtys, Decimal("0")) db.add(ShoppingItem( ingredient_name=name, quantity=total, unit=unit, is_purchased=False, ))

Example: "Week of Pasta"

Spaghetti Bolognese × 2 + Pasta Primavera = combined pasta quantity in one line item. Oregano gets flagged as pantry item.
